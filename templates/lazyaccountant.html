The Lazy Accountant

This project was inspired by one of the topics from the book, Automate the Boring Stuff with Python by Al Sweigart. Initially the task in the book was to extract e-mail addresses and telephone numbers from a pdf document. This is where I got introduced to the topic of regular expressions.
I immediately realized that using the same methodology I can automate one of my daily task, which is to identify payments from various bank statements using the invoice numbers as reference.

First I broke down my process and commented the steps I need python to do for me:

# Open pdf or excel etc. and copy all to clipboard
# Create a regex for numbers
# Get the text off the clipboard
# Extract the number from this text
# Open excel and insert results Column A
# Create excel and insert extracted numbers to Column A
# Print done when done

I was able to dedicate a few hours daily and my weekends to work on this code and after 2 weeks of headache I had a working solution, which looks similar as shown below.
Keep in mind that I had to edit out sensitive informations from my code due to security reasons.

I am running this code with a batch file so instead checking hundreds of pages I simply copy the payment details into the clipboard and run the program. (Coffee time!)

Doing this project really worth it. Setting up my environment, downgrading/upgrading modules, regular expressions, openpyxl.. but most importantly the workflow & problem solving part was truly enjoyable.
Later, I also set up my file management within O365 using SharePoint, Power BI and Flow so almost everything is automated.

Check out the code also on github. Cheers!
